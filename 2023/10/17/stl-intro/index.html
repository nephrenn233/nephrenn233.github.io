常见的 STL 在算法竞赛中的用法

Author: Nephrenn Date: 十月 17, 2023  19:14:34 Category: ACM/XCPC

STL (Standard Template Library) 提供了一系列常用的算法与数据结构。下面对几种常用算法和容器的用法进行说明。

容器

vector

vector 可以理解为变长数组，定义在头文件 <vector> 中，支持使用下标随机访问，不支持在任意位置 $O(1)$ 插入。

用法

下面是 vector 的使用方法

遍历 vector 可以使用下标或迭代器。迭代器相当于是 STL 容器的指针，用 * 解除引用。

某些函数对于 STL 容器是通用的，例如 size() empty() begin() end() 等等，之后不再讲解

实例

我们可以使用 vector 代替邻接表存图。
Luogu B3643

stack & queue

栈和队列，分别定义在 <stack> 和 <queue> 中。插入和弹出元素时与他们的性质一样，只能在头部（尾部）插入，在头部（头部）弹出。同样也可以使用迭代器遍历。

用法

实例

Luogu P1449

Luogu P1996

deque

双端队列，定义在 <deque> 中。与普通队列的区别是可以在头部和尾部插入或弹出元素。

用法

priority_queue

优先队列，相当于大根二叉堆，定义在 <queue> 中。可以通过重载运算符 < 或仿函数的方式改变比较规则，从而实现小根堆等。

用法

实例

裸模板 Luogu P1090
可以用于实现堆优化的 Dijkstra 最短路算法 Luogu P4779
对顶堆，动态维护第 $k$ 大数 Luogu P1801

set & multiset & unordered_set && unordered_multiset

set 实现了有序集合，内部元素不允许有重复且按一定规则排序（需定义小于号）。
multiset 相当于允许有重复元素的 set，两者都定义在 <set> 中。
unordered_set 和 unordered_multiset 对应无序的 set 和 multiset，定义在 unordered_set 中。
set 插入元素和迭代器增减的复杂度均为 $O(logn)$

用法

四个容器的用法大同小异，下面以 set 为例

pair

包含两个元素的模板类型，定义在 <utility> 中。默认以第一关键字比较。

用法

map & unordered_map

类似于哈希表，分别在 <map> 和 <unordered_map> 中。
map 中储存的都是 pair，同时拥有键值和实值，不允许两个元素拥有相同的键值。
map 会根据键值对元素进行从小到大的排序，而 unordered_map 无序。
map 访问元素的复杂度为 $O(logn)$

用法

实例

Luogu P1102

bitset

相当于一个多位二进制数，在 <bitset> 中。支持位运算操作。

用法

实例

Luogu P7076

算法(?)

以下几个函数均作用在序列上，范围为左闭右开，用法类似。

sort()

在 $O(nlogn)$ 的时间内完成排序。

用法

random_shuffle()

随机打乱一段序列。

unique()

去重并返回尾迭代器（末尾元素的下一个位置的迭代器）

lower_bound() & upper_bound()

在有序序列上二分查找。
lower_bound(v.begin(), v.end(). x) 返回第一个大于等于 $x$ 的元素的迭代器，没有则返回 v.end()
upper_bound(v.begin(), v.end(). x) 返回第一个大于 $x$ 的元素的迭代器，没有则返回 v.end()
